安裝使用 eslint

發表於 2020-06-16 更新於 2020-07-25 分類於 JavaScript

在多人一起撰寫一個專案的時候，因為每個人撰寫的規範不同，程式碼會很亂。因此我們可以使用 eslint 來統一規範格式

  1. 在安裝 eslint 之前，我們要先專案底下建立一個 package.json 檔案出來，使用 npm init 建立

  1. 然後安裝 eslint 的全域環境 (如果之前有安裝過可以略過)，使用 npm install -g eslint 來安裝全域的環境

  2. 全域環境安裝完成之後，接著我們可以使用 eslint --init 來對環境進行初始化

  3. 初始化之後會出現一些問題，我們可以選擇相對應的設定 (紅色為建議選項)

  • How would you like to use ESLint? (Use arrow keys)

    To check syntax only
    To check syntax and find problems
    To check syntax, find problems, and enforce code style

  • What type of modules does your project use? // 專案中的模組管理是使用哪種類型

    JavaScript modules (import/export)
    CommonJS (require/exports)
    None of these

  • Which framework does your project use? // 專案使用哪種框架

    React
    Vue.js
    None of these

  • Does your project use TypeScript? (y/N) // 專案有沒有使用 TypeScript

  • Where does your code run? // 你的 code 在哪裡運行

    (*) Browser
    ( ) Node

  • How would you like to define a style for your project? // 怎麼定義你的 coding style 在你的專案上?

    Use a popular style guide // 使用主流的 coding style
    Answer questions about your style // 透過問題來選擇 coding style
    Inspect your JavaScript file(s) // 檢查自己的 JS 檔案來快速產生 coding style

  • Which style guide do you want to follow? // 選擇哪一種主流的?

    Airbnb: https://github.com/airbnb/javascript
    Standard: https://github.com/standard/standard
    Google: https://github.com/google/eslint-config-google

  • What format do you want your config file to be in? // 選擇哪一種格式來做儲存?

    JavaScript
    YAML
    JSON

  • Would you like to install them now with npm? (Y/n) // 同時是否要安裝相對應的套件?

(Y)
