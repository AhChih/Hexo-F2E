ASI 自動插入分號

發表於 2020-06-01 更新於 2020-07-26 分類於 JavaScript

程式碼1

那如果我們把 ‘叫我小明’ 退到上一行 return 的後方

你會發現 ‘叫我小明’ 就會正常的顯示出來了

為什麼中間的斷行會發生錯誤呢 ?

ASI

這個我們稱為 ASI

當 JavaScript 語句沒有加上分號時，會受到自動插入分號 ASI 規則影響

像剛剛會出現 undefiend 的結果

這段程式碼其實會像這樣

它會在 return 的後方自動加上分號

因為這樣就沒有辦法正確的回傳 ‘叫我小明’;

但最後的結果會是 2

程式碼3

你會看到這段的結果還是 2，原因是它會透過 ASI 去執行

程式碼4

雖然 ASI 有優點，但也有缺點

這個程式碼呢，我們稱為立即函式

這段程式碼會先在前面插入一段匿名函式
